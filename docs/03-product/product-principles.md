# Sentiq Product Principles

Version: 1.0

---

# Purpose

This document defines the principles that guide every product decision inside Sentiq.

Features change.

Interfaces evolve.

Technology changes.

Principles remain.

Every new idea, feature or improvement should be evaluated against these principles.

---

# Principle 1

# Development Over Measurement

Sentiq does not exist to measure athletes.

Sentiq exists to help athletes develop.

Data is useful only when it creates improvement.

A metric without action is just information.

Every measurement should answer:

"What can the athlete do better because of this?"

---

# Principle 2

# Reflection Creates Growth

Reflection is the foundation of the product.

Athletes improve when they understand:

- What happened.
- How they responded.
- What they learned.
- What they want to improve.

Every interaction should encourage awareness.

---

# Principle 3

# Small Actions Create Big Results

Sentiq should never require significant effort.

The product wins when it becomes part of the athlete's routine after training and matches.

A quick evaluation repeated consistently creates more value than a complex activity completed once.

---

# Principle 4

# Reduce Friction

Every additional step is a barrier.

Every unnecessary question reduces engagement.

Every complex flow reduces consistency.

Before adding anything, ask:

Can this be simpler?

---

# Principle 5

# One Primary Action

Every screen should have a clear purpose.

The user should immediately understand:

"What should I do here?"

Avoid:

- Multiple competing actions.
- Excessive choices.
- Information overload.

---

# Principle 6

# Value Before Data

Sentiq should provide value before asking for information.

The athlete should feel:

"I received something useful."

Not:

"I had to complete a form."

---

# Principle 7

# Progress Over Perfection

Athlete development is a journey.

The product should reward:

- Consistency.
- Improvement.
- Learning.
- Effort.

Never punish:

- Mistakes.
- Bad days.
- Lack of perfection.

---

# Principle 8

# Personal Growth, Not Comparison

Every athlete has a different journey.

Sentiq should compare athletes against themselves.

Never create unnecessary external comparison.

Avoid:

- Rankings.
- Emotional leaderboards.
- Public scores.
- Competitive pressure around personal development.

The competition is with yesterday's version of yourself.

---

# Principle 9

# The Athlete Owns Their Journey

The athlete is the owner of their development.

Sentiq provides guidance.

AI provides suggestions.

Coaches provide support.

But the athlete decides.

---

# Principle 10

# AI Assists, Humans Lead

Artificial intelligence is a tool.

Not the product.

Not the coach.

Not the decision maker.

Coach Sentiq is not a chatbot. It generates specific outputs: objective suggestions, weekly preparation, monthly reports, and Learning Cards.

AI should:

- Suggest.
- Explain.
- Encourage.
- Help reflect.

AI should never:

- Diagnose.
- Judge.
- Replace coaching.
- Make decisions about the athlete.
- Present itself as a conversational assistant.

---

# Principle 11

# Coaches Are Partners

Sentiq exists to strengthen coaches.

Never to replace them.

Every coach-facing feature should improve:

- Communication.
- Understanding.
- Follow-up.
- Development conversations.

---

# Principle 12

# Emotional Intelligence Without Clinical Language

Sentiq works with emotions because emotions affect performance.

However, Sentiq is not a clinical product.

The objective is:

Awareness.

Reflection.

Growth.

Not diagnosis.

---

# Principle 13

# Immediate Value

The athlete should experience value quickly.

A new user should not need days of setup before understanding Sentiq.

The first evaluation should create curiosity.

The first week should create consistency.

The first month should create visible progress through IEA.

---

# Principle 14

# Design For Trust

Athletes must trust Sentiq.

Trust comes from:

- Transparency.
- Simplicity.
- Respect.
- Privacy.
- Honest communication.

Never manipulate.

Never create artificial urgency.

Never exploit emotions.

---

# Principle 15

# Avoid Feature Inflation

More features do not automatically create more value.

Every feature has a cost:

- Development cost.
- Maintenance cost.
- Cognitive cost.
- Learning cost.

The question is not:

"Can we build this?"

The question is:

"Should we build this?"

---

# Feature Evaluation Framework

Before building a feature, answer:

## Problem

What user problem are we solving?

---

## User

Who benefits?

Athlete?

Coach?

Club?

---

## Value

What changes after using it?

---

## Frequency

Will users need this regularly?

---

## Simplicity

Can the same value be achieved easier?

---

## Measurement

How will we know it worked?

---

# Things Sentiq Optimizes For

Sentiq optimizes for:

- Retention.
- Evaluation consistency.
- Athlete development.
- Meaningful engagement.
- User trust.
- Product love.

---

# Things Sentiq Does Not Optimize For

Sentiq does not optimize for:

- Addiction.
- Time spent in app.
- Notifications opened.
- Vanity metrics.
- Feature quantity.

---

# The Ultimate Product Question

Before any decision:

"Does this help athletes become better versions of themselves?"

If yes:

Explore it.

If no:

Question it.
