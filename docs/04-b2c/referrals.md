# Sentiq B2C Referral Strategy

Version: 1.1

Status: Planned referral program + Current related mechanisms clearly separated

---

# Purpose

This document defines how Sentiq approaches athlete-to-athlete growth.

The objective is organic growth through athletes who believe Sentiq helps them improve and want to share it with others.

---

# Referral Philosophy

The strongest recommendation is not:

"Download this app."

The strongest recommendation is:

"I am using this to become a better athlete, and I think it can help you too."

Referrals should be based on shared development, not artificial incentives.

---

# Critical Distinction

Sentiq currently has several invite-like mechanisms. Only one of them is a true referral program, and that one is still planned.

| Mechanism | Status | What it does | Attribution | Pro reward for inviter? |
|-----------|--------|--------------|-------------|-------------------------|
| Red follow invites | Current | Invite someone to follow / be followed | Social graph only | No |
| Campaign codes | Current | Shared partner/event code → Pro grant | Campaign, not athlete | No (recipient grant) |
| Org invitations | Current | Club onboards athletes | Organization | Possible org Pro grant for member |
| Early Adopters | Current | First-100 program | Program enrollment | Lifetime Pro for enrollee |
| Athlete referral program | Planned | Personal invite link/code | Referring athlete | Possible quality-based incentive |

Do not call Red invites, campaign codes or org invites "referrals" in product copy or analytics without clarifying the difference.

---

# Current: Red Invites (Not Referrals)

Athletes can:

- Search / invite by user or email
- Accept or reject invitations
- Build followers / following
- Share evaluations (privacy permitting)
- Suggest objectives to followed athletes

Home surfaces:

- Pending invitation banner
- CTA when the athlete has no followers

Red strengthens community and retention.

Red is not an acquisition attribution system and does not grant Pro for inviting.

---

# Related: Campaigns (Not Referrals)

Campaigns use a shared code belonging to a partner, club pilot or event.

Characteristics:

- One code, many recipients
- Capacity limits
- Benefit / Pro grant to the joiner
- Admin-managed in Sentiq Org super-admin
- Redeemed via campaign landing flow

Campaigns are a powerful acquisition and distribution tool.

They are partner-led, not athlete-to-athlete referrals.

See [campaigns.md](./campaigns.md).

# Planned: Athlete Referral Program

## Strategic Objective

When built, the referral system should:

- Acquire new athletes
- Reduce acquisition cost
- Strengthen community
- Turn advocates into measurable growth channels

## Best Moments To Ask

After value:

- Multiple evaluations completed
- Objective achieved
- Meaningful insight / report
- Visible IEA progress

Avoid:

- Immediately after registration
- Before the athlete understands value
- During frustration

## Planned Methods

- Personal invitation from the app
- Personalized referral link (example shape: `sentiqapp.com/join/{handle}`)
- Optional simple codes for ambassadors / events (distinct from campaign benefit codes)

## Planned Incentive Philosophy

Rewards should reinforce community, not create spam invites.

Possible later incentives:

- Recognition / ambassador status
- Limited Pro time for quality activations
- Community experiences

Avoid:

- "Invite 5 friends to unlock core features"
- Rewarding invites without activation/retention quality

## Planned Metrics

- Invites sent
- Invite → registration conversion
- Referred activation (first evaluation / objective)
- Retention of referred users vs organic
- Abuse / low-quality invite rate

---

# Club And Ambassador Models (Strategy)

Clubs can amplify growth through Org + campaigns + athlete word of mouth.

Ambassadors can share journeys and invite athletes.

These models may use campaigns or future referral links, but should keep authenticity.

---

# Messaging

Good:

"I started using Sentiq to improve as an athlete. Join me."

Avoid:

"Download this app."

The athlete should feel they are helping someone improve, not promoting a product.

---

# Risks

- Incentive abuse
- Low-quality users
- Loss of authenticity
- Confusing analytics by mixing Red / campaigns / referrals

---

# Future Opportunities (After Core Referral Exists)

- Team challenges
- Participation-based club recognition (not performance ranking)
- Athlete milestones
- Ambassador programs

---

# Final Principle

The best referral program for Sentiq is not based on rewards.

It is based on athletes helping other athletes grow.

Until the planned program ships, document and measure Red, campaigns and org invites as separate systems.
